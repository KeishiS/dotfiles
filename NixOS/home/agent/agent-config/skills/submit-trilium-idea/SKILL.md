# TriliumNextへのアイデア送信

アイデアをMarkdownノートとして統一した形式で保存する。
送信は外部状態を変更するため、ユーザーが保存または送信を依頼した場合だけ実行する。

## 保存仕様

- 親ノート: `Idea Inbox`
- ノート種別: `code`
- MIMEタイプ: `text/markdown`
- タイトル: 内容を一文で識別できる簡潔な表現
- 本文: 下記テンプレートに従う
- ラベル:
  - `idea`（値なし）
  - `status=inbox`
  - `source=mcp`

本文でユーザーの情報を補完するときは、事実と推測を混同しない。
情報がない節には内容を創作せず、「未記入」または該当する未決事項を記す。

## 本文テンプレート

```markdown
## 概要

アイデアを短く要約する。

## 背景・課題

着想の背景と解決したい課題を記す。

## 提案

実現したい内容を具体的に記す。

## 期待する効果

- 期待できる効果を記す。

## 懸念・未決事項

- 不明点、制約、判断が必要な事項を記す。

## 次のアクション

- [ ] 次に行う調査または作業を記す。
```

タイトルはTriliumNextのノートタイトルとして保持されるため、本文に同じ見出しを
重複させない。不要な節は削除せず、短く「該当なし」または「未記入」とする。

## 送信手順

1. 利用可能なMCPツールの入力スキーマを確認する。引数名を推測しない。
2. `triliumnext_resolve_note_id` で `Idea Inbox` を完全一致優先で検索する。
3. 一意に特定できない場合は候補を示してユーザーに確認し、送信しない。
4. 保存前にタイトル、Markdown本文、親ノートをユーザーへ簡潔に提示する。
   ユーザーが同じ依頼内で内容と送信を明示済みなら、重ねて確認する必要はない。
5. `triliumnext_create_note` で `code`、`text/markdown` のノートを作成する。
6. `triliumnext_read_attributes` で既存属性を確認してから、
   `triliumnext_manage_attributes` で所定のラベルを重複なく設定する。
7. 作成結果を確認し、タイトル、note ID、親ノート、設定したラベルを報告する。

## 更新時の規則

- 既存ノートの変更は、ユーザーが対象ノートと変更内容を指定した場合だけ
  `triliumnext_update_note` で行う。
- 更新前に `triliumnext_get_note` で現在の内容を取得し、無関係な内容を保持する。
- 属性変更前に `triliumnext_read_attributes` で現在値を確認する。
- `delete_note`は使用しない。
- APIトークン、OAuthトークン、パスワードなどの秘密情報を本文や属性へ保存しない。

## 失敗時

一部だけ成功した場合は再実行でノートを重複作成しない。作成済みのnote IDを使って
不足する属性だけを設定する。利用可能なツール、親ノート、認証のいずれかが不足する
場合は、実行済みの操作と未完了の操作を区別して報告する。
